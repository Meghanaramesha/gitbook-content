## How can iOS 14+ privacy changes affect user acquisition campaigns?

‚Äú‚Ää*I don‚Äôt see data on iOS! (in the first 72h)‚Äù  
‚ÄùCPIs are really high, there must be something wrong?!‚Äù*

Well, yes & yes. You are not able to see any installs coming through in the first 72 hours thanks to the privacy changes that happened on iOS. Installs are happening in the meantime, you are just not able to see them. Don‚Äôt worry, you won‚Äôt lose any data.

You just need to be prepared for something like this will happen:

![](https://substackcdn.com/image/fetch/w_1456,c_limit,f_auto,q_auto:good,fl_progressive:steep/https%3A%2F%2Fsubstack-post-media.s3.amazonaws.com%2Fpublic%2Fimages%2Ff09ccffc-27b3-4108-9927-ff50c770bc00_1610x878.png)

What can you do?

Don‚Äôt make any changes in the first 72h => 0 installs = Facebook is not optimized at all. Wait until you get data, then make changes!

Also, the privacy threshold on¬†**Facebook is 88 installs per day per campaign**. As you can see, the campaign above was not anywhere close to passing that threshold, which means we also don‚Äôt see the full picture in terms of postbacks. What can be done here?
